## 摘要
脑电图基础模型越来越依赖于多数据集训练和评估，然而公开的脑电图数据集仍然缺乏一个共享的任务规范层，可以将异构记录转化为可重复使用的基准单元。现有标准组织文件、元数据和来源，但并未在统一语言和规则书下指定脑电图任务，导致关键任务语义分散在论文、代码和人工解释中。我们研究了是否可以通过结构化的任务规范语言配以共享规则书来标准化异构的公开脑电图数据集。我们的方法将每个基准条目表示为一个与可执行任务内核同步的任务文档，规则书定义任务字段、证据要求、文档-内核对齐、审查状态和机器可检查的约束。使用这种方法，我们发布了一个经过社区审查的脑电图基准语料库，包含53个已完成和审查的条目，具有245个跨越多种范式的任务定义，并引入了NeuroDoc和NeuroAudit作为操作支持层，用于规则书引导的起草、升级、审查、修订和发布管理。我们进一步考察了生成的基准单元是否可以在四个脑电图基础模型骨干的共享下游设置中实例化，为可重用、可审计和可执行的脑电图基准测试基础设施提供了基于执行的证据。

## Abstract
Electroencephalography (EEG) foundation models increasingly rely on multi-dataset training and evaluation, yet public EEG datasets still lack a shared task specification layer that can turn heterogeneous recordings into reusable benchmark units. Existing standards organize files, metadata, and provenance, but they do not specify EEG tasks under a common language and rulebook, leaving critical task semantics scattered across papers, code, and manual interpretation. We investigate whether heterogeneous public EEG datasets can be standardized through a structured task specification language paired with a shared rulebook. Our methodology represents each benchmark entry as a task document synchronized with an executable task kernel, with the rulebook defining task fields, evidence requirements, document-kernel alignment, review states, and machine-checkable constraints. Using this methodology, we release a community-reviewed EEG benchmark corpus centered on 53 completed and reviewed entries with 245 task definitions spanning diverse paradigms, and we introduce NeuroDoc and NeuroAudit as the operational support layer for rulebook-guided drafting, upgrading, review, amendment, and release management. We further examine whether the resulting benchmark units can be instantiated in a shared downstream setting across four EEG foundation model backbones, providing execution-based evidence for reusable, auditable, and executable EEG benchmarking infrastructure.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：随着脑电图（EEG）基础模型越来越依赖多数据集训练和评估，公开的EEG数据集仍缺乏一个**共享的任务规范层**，导致异构的原始记录无法转化为可复用的基准测试单元。现有的数据标准（如BIDS、EEG-BIDS）只组织文件、元数据和溯源，而未能在统一语言和规则书下定义EEG任务。任务语义信息分散在论文、代码和人工解释中，跨数据集研究需要大量未文档化的手动解释，成为可重复基准构建的主要瓶颈。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：构建一个**基于条目的表示**，每个基准条目由两部分组成：
  - **任务文档（Document）**：人类可读的任务规范，包含范式、文件扫描结果、任务定义（类型、分段方法、标签路径、元级别）、数据质量说明和内核实录等。
  - **任务内核（Kernel）**：可执行的、数据集特有的代码逻辑，将文档中的规范在统一处理管线中实例化。
- **任务规范语言的四个耦合层**：
  1. **任务类型**：分类或回归。
  2. **分段方法（Epoch Method）**：事件锁定、长事件锁定或连续窗口。
  3. **标签路径（Label Route）**：事件(event)、辅助通道(misc)或文件属性(info)。
  4. **元级别（Meta Level）**：原始(raw)或超(super，需要进一步推导)。
- **规则书（Rulebook）**：核心治理基础，定义证据优先级、任务声明的可接受性、文档-内核对齐、审查状态（wait_kernel_update, wait_manual_review, completed, hallucination）以及可机器检查的约束（如字段值合法范围、事件一致性、分类任务类数≥2等）。规则书既是规范性的（面向审查者），也是可编程的（通过detect.py实施机器检查）。
- **工具链**：

- **充分性评估**：
  - **客观**：基准构建过程遵循明确规则书，审查状态由社区控制，执行验证使用公开骨干和标准超参数。
  - **公平**：不同模型在相同任务定义、相同预处理管线（通过内核统一）下比较，排除了手动调整的偏差。但论文也承认执行验证局限于已完成的53个条目，更广泛的数据库中还有约200个草稿未用于分析。
  - **局限**：未进行广泛的消融实验（如去掉规则书的效果、不使用LLM辅助的影响等）。生成质量评估仅针对NeuroDoc的一个版本（Kimi-K2.6）在43个数据集上，可能偏向于结构化良好的数据集。

## 6. 论文的主要结论与发现

- **主要结论**：提出的**规则书引导的任务规范层**（NeuroDoc + NeuroAudit + 基准条目）能够将异构EEG数据集转化为**可执行、可审计、可维护的基准单元**，且无需重新打包原始记录。通过四个大模型骨干的实例化验证，证明了该基础设施的下游可用性。
- **具体发现**：
  - 基准语料库覆盖广泛的任务类型和范式，任务定义在分段方法、标签路径、元级别上具有多样性。
  - 执行验证显示所有四个骨干均能在统一任务定义下运行，Macro-F1在任务家族间分布合理，被试内评价显著优于跨被试（平均差距+12.6个百分点），随着类别数增加性能下降（log2类斜率-0.14）。
  - NeuroDoc生成的文档质量因数据集来源而异：OpenNeuro/BIDS最佳，PhysioNet和TUH最弱（后两者存在间隔侧边栏处理、外部标注发现等挑战）。
  - 文档-内核对齐是挑战：仅47.6%的内核验证成功，但通过升级管线可以修复。

## 7. 优点：方法或实验设计上的亮点

- **系统化程度高**：首次提出完整的EEG任务规范语言+规则书+工具链，将基准构建从手工操作提升为结构化、半自动化的流程。
- **可执行性**：文档与内核同步，确保规范可直接实例化，避免了传统仅文本描述的歧义。
- **审计机制**：NeuroAudit提供基于规则书的社区审查接口，审查状态透明，支持机器检查约束，确保基准质量。
- **可扩展性**：规则书是可演化的，社区可以提议扩展字段或添加新任务模式，支持向后兼容的废弃机制。
- **大规模实践**：基于超过17.7TB的扫描数据和约8.46GB的参考资源，生成了53个高质量条目，验证了方法的实际可行性。
- **公平性考虑**：不重新打包原始EEG数据，遵守数据使用条款，通过任务规范层而非数据重分发促进基准复用。

## 8. 不足与局限

- **实验覆盖有限**：定量分析仅限于53个已完成条目（约200个草稿未分析），执行验证覆盖18个数据集，可能不足以代表所有EEG范式的多样性。
- **生成质量依赖数据集结构**：对于非BIDS、自定义格式或临床数据集（如TUH、PhysioNet），NeuroDoc生成质量较差，事件提取和内核验证成功率低（分别为16.7%和47.6%的总体成功率）。
- **LLM幻觉风险**：LLM辅助起草可能引入过度解释或不准确的证据，虽然规则书和人类审查可以缓解，但不能完全消除。
- **计算资源门槛**：虽然基准构建主要是CPU友好的，但执行验证仍需多GPU资源，可能限制小实验室的参与。
- **通用性限制**：当前任务规范语言仅支持分类和回归，不支持多标签、序列预测等复杂任务；标签路径只支持三种（事件、辅助通道、文件属性），可能不覆盖所有EEG任务的需求。
- **社区治理挑战**：规则书更新和条目升级机制需要社区持续维护，实际长期可行性未验证。
- **偏差风险**：语料库继承原始数据集的偏差（如人口统计不均、实验室特定协议），不应过度解读为临床或泛化性证据。
